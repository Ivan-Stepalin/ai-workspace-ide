WebSocket-протокол

Создание / переподключение PTY
// Клиент → Сервер
{ type: 'terminal_create',
  projectId: 'proj-abc',
  agent: 'coder',   // опц.: роль
  cols: 220, rows: 50,
  terminalId: 'tid-xyz' } // переподключение

// Сервер → Клиент
{ type: 'terminal_ready',
  terminalId: 'tid-xyz' }
// затем поток буферизованных данных...
Поток данных
// Клиент → Сервер (ввод)
{ type: 'terminal_input', data: 'ls\r', terminalId: ... }

// Сервер → Клиент (вывод)
{ type: 'terminal_data', data: 'file1.ts\r\n...', ... }

// Изменение размера
{ type: 'terminal_resize', cols: 180, rows: 45, ... }
Бродкасты (главный WS)
// Дерево файлов обновилось
{ type: 'tree_updated', projectId: ... }

// Файл изменён
{ type: 'file_changed',
  projectId: ..., path: 'src/app.ts' }

// Список проектов изменился
{ type: 'projects_updated' }
Жизненный цикл PTY
terminal_create → spawn PTY (bash / claude)
Буфер вывода (256 КБ cap)
WS разрывается → PTY продолжает жить
30 мин бездействия → GC убивает PTY
terminal_close → мгновенное уничтожение

Режим разработки

Backend

# Из корня монорепо:
npm run dev:backend
# = cd backend && npm start
# = tsc && node dist/index.js

# Или вручную:
cd backend
npx tsc          # компиляция TS → dist/
node dist/index.js  # запуск (:3001)
Нет авто-перезагрузки. После любой правки в backend/src/** — повторить tsc и перезапустить процесс вручную.

Frontend

# Из корня монорепо:
npm run dev:frontend
# = cd frontend && npm run dev
# = vite dev → :5173 (HMR)
  • Изменения подхватываются мгновенно через HMR
  • Vite проксирует /api/* и /ws на :3001 — CORS не нужен
  • TypeScript-ошибки видны в терминале и браузере

Оптимизации сборки

Lazy-загрузка чанков

// vite.config manualChunks:
'react':        ['react', 'react-dom']
'xterm':        ['@xterm/xterm']
'editor.api':   Monaco core
'ts.worker':    TS language server
'editor.worker': base worker

Monaco и xterm грузятся только при первом открытии файла / терминала

Начальная загрузка

~90 КБ
gzip (index + react)
  • Monaco (~12 МБ) — отдельный lazy-чанк
  • xterm — отдельный lazy-чанк
  • Грузятся по требованию, кэшируются навсегда

Кэширование

ПутьCache-Control
/assets/*immutable, 1 год
/index.htmlno-cache
/sw.jsno-cache
остальное→ index.html

Хэши в именах файлов = автоматическая инвалидация при новой сборке

PWA и офлайн-режим

  • Service Worker precache — всё включая Monaco-воркеры
  • Лимит precache поднят (maximumFileSizeToCacheInBytes)
  • После первой загрузки — IDE работает без интернета
  • HTTPS на LAN: mkcert + frontend/certs/ → Vite подхватывает автоматически

Monaco без CDN

  • Пакет monaco-editor в зависимостях — нет внешних запросов
  • Web Workers собираются через ?worker (Vite) как отдельные чанки
  • TypeScript, JSON, CSS, HTML language servers — всё локально
  • Работает в изолированных сетях и корпоративных окружениях

Monaco Editor — локальная сборка

Проблема CDN

  • Зависимость от внешней сети
  • Медленный первый старт
  • Проблемы с PWA/офлайн
  • CORS для web-воркеров

Решение: локальный бандл

  • Monaco bundled как npm-пакет
  • Lazy-chunk (не блокирует старт)
  • Web Workers как ?worker (Vite)
  • Кэшируется браузером навсегда
  • Работает офлайн

Чанки при сборке

// vite.config manualChunks:
'editor.api': // Monaco core API
  ['monaco-editor/esm/vs/editor/editor.api']

'ts.worker': // TypeScript language server
  ['monaco-editor/...typescript/ts.worker']

'editor.worker': // Base editor worker
  ['monaco-editor/...editor.worker']

'react': ['react', 'react-dom']
'xterm': ['@xterm/xterm', ...]

только index + react ≈ 90 КБ gzip

Система агентов

Агент = интерактивный claude CLI в PTY-терминале. Роль задаётся флагом --append-system-prompt

Overseer

Общий менеджер. cwd = PROJECTS_DIR. Видит все проекты. Код не правит — рекомендует нужного агента.

Manager

Декомпозирует задачи. Координирует работу команды. Планирует итерации.

Coder

Пишет код, создаёт файлы, делает git-коммиты. Основной рабочий агент.

Reviewer

Code review: качество кода, баги, безопасность. Анализирует, предлагает улучшения.

Как запускается агент

claude \
  --append-system-prompt "[роль из PROMPTS]" \
  --cwd "/path/to/project"
# ↑ интерактивный PTY (видны мысли, вызовы инструментов)

Плюсы и минусы
Что решает, что не решает

Какую проблему решает

❌ Без IDE

  • PTY-сессия привязана к процессу терминала — обрыв соединения = потеря сессии
  • Нет API для управления агентами: запустить, переподключиться, получить буфер
  • Файлы, терминал и git — разные инструменты без общего состояния
  • Claude CLI требует локального окружения — недоступен через браузер удалённо
  • Нет персистентного хранилища сессий между подключениями
  • Среда разработки не воспроизводима по URL без дополнительной инфраструктуры

✓ С AI Workspace IDE

  • Открыл браузер → всё готово к работе
  • Сессии агентов живут на сервере, не в браузере
  • Файлы + терминал + git + агент в одном окне
  • Доступ с любого устройства по IP + Telegram-бот
  • Каждый проект — отдельная вкладка браузера
  • Офлайн через PWA, Monaco работает без CDN
Ключевая идея: IDE крутится на сервере, браузер — просто дисплей. Закрыл вкладку → агент продолжает работать. Переоткрыл → подключился обратно.

Ключевые архитектурные решения

Сессии на сервере, не в браузере

PTY живёт на сервере. Браузер — только дисплей. Закрыл вкладку → агент работает дальше. Переоткрыл → переподключился за счёт буфера и wsId. Это критично для долгих AI-задач.

Все вкладки смонтированы одновременно

Вместо умонтирования скрытых вкладок — display:none. Агент в фоновой вкладке не выгружается. Ценой небольшого потребления памяти — надёжные фоновые процессы.

Один воркспейс — одна вкладка

Упрощает код кардинально: нет переключения проектов, нет условных хуков, нет конфликтов состояний. «Другой проект» = новая вкладка браузера.

Monaco без CDN

Полная независимость от сети. PWA precache включает Monaco-воркеры. Первый старт — только 90 КБ, Monaco грузится lazy при открытии файла.

Автообнаружение проектов

Любая папка в PROJECTS_DIR автоматически становится проектом. Агент создал папку → проект появился в IDE без перезапуска.

memo + стабильные колбэки

FileTree и TerminalPanel обёрнуты в memo. Ввод в Monaco не вызывает ни одного ререндера терминала. Колбэки через ref — не зависимости эффекта.

Безопасность и ограничения

⚠️ Важные ограничения

  • Нет аутентификации пользователей по умолчанию — рассчитан на доверенную сеть
  • Любой имеет доступ к файловой системе через API
  • Claude CLI выполняет произвольный код на сервере
  • Telegram: без whitelist — любой может отправить команду

Рекомендации для прода

  • Запускать за VPN / nginx с auth_basic
  • HTTPS (mkcert для LAN)
  • TELEGRAM_ALLOWED_CHAT_IDS — whitelist
  • PROJECTS_DIR — в отдельном разделе
  • Ограничить CPU/RAM через systemd cgroup

Что защищено

  • TELEGRAM_ALLOWED_CHAT_IDS — whitelist чатов
  • Данные не в git (строгий .gitignore)
  • Нет хардкода путей и IP в коде
  • Файловое дерево фильтрует node_modules, .git
  • Claude CLI sandboxing (встроенные ограничения)

Env-переменные не попадают в frontend

# backend/.env — только серверные
TELEGRAM_BOT_TOKEN=secret
DB_PATH=/secret/path

# frontend — только VITE_* переменные
# (публичные, попадают в bundle)
VITE_BACKEND_HOST=192.168.1.100
