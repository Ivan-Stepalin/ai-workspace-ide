# AI Workspace IDE

Браузерная IDE с агентами на базе Claude, интегрированными терминалами, файловым деревом, редактором Monaco и git. Управляет отдельными проектами: в каждом можно открывать сколько угодно сессий агентов (менеджер / кодер / ревьюер) и терминалов, запускать приложение проекта одной кнопкой. Есть **общий менеджер**, который видит все проекты сразу, отвечает на кросс-проектные вопросы и добавляет новые репозитории по ссылке.

```
ai-workspace-ide/
├── backend/    # Node + Express + WebSocket: проекты, git, PTY-терминалы, агенты (claude CLI)
└── frontend/   # React + Vite: IDE-интерфейс
```

Пользовательские данные (проекты и БД) хранятся **вне** репозитория — см. `DATA_DIR` ниже.

## Требования

- Node.js 20+
- [Claude CLI](https://docs.claude.com/claude-code) в `PATH` (бэкенд вызывает `claude` для агентов)
- `python3` (фолбэк-статика для проектов без скрипта запуска)

## Установка

```bash
npm run install:all
# или вручную:
#   npm --prefix backend install
#   npm --prefix frontend install
```

## Настройка

Скопируй примеры окружения и при необходимости поправь:

```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
```

- **backend/.env**
  - `PORT` — порт бэкенда (HTTP + WebSocket), по умолчанию `3001`.
  - `DATA_DIR` — где хранить данные (создаются `projects/` и `workspace.db`), по умолчанию `./data`.
    Можно переопределить точечно: `PROJECTS_DIR`, `DB_PATH`.
- **frontend/.env**
  - `VITE_BACKEND_HOST` / `VITE_BACKEND_PORT` — где искать бэкенд. По умолчанию хост берётся
    из адреса, по которому открыт фронтенд, так что для localhost править не нужно.

## Запуск (dev)

В двух терминалах:

```bash
npm run dev:backend     # tsc + node dist/index.js на :3001
npm run dev:frontend    # vite dev-сервер
```

Открой адрес, который покажет Vite.

## Сборка

```bash
npm run build           # собирает backend (tsc) и frontend (vite build)
npm run start:backend   # запуск собранного бэкенда
```

## Использование

### Проекты и где они хранятся

Каждый проект — это отдельная папка внутри `PROJECTS_DIR` (по умолчанию `<DATA_DIR>/projects`).
Все папки в этом каталоге **автоматически** становятся проектами и появляются в списке вверху —
их не нужно регистрировать вручную.

Создать проект — кнопка `+` рядом со списком проектов. Переключение — клик по имени проекта.

### Агенты (сессии)

В контексте выбранного проекта в панели **ДЕЙСТВИЯ** можно открыть сколько угодно независимых
сессий агентов:

- **🤖 Менеджер** — декомпозирует задачи, координирует;
- **🤖 Кодер** — пишет код, создаёт файлы, делает коммиты;
- **🤖 Ревьюер** — проверяет качество и безопасность.

Каждая сессия — отдельная вкладка со своей историей диалога. Закрытие вкладки завершает сессию
на бэкенде (процесс агента убивается, история чистится).

### Общий менеджер

Кнопка **🧭 Общий менеджер** (секция **ГЛОБАЛЬНО**) открывает кросс-проектную сессию. Он:

- видит **все** проекты сразу (структура, последние коммиты, выжимка README подкладываются ему в контекст);
- отвечает на общие вопросы — что в каком проекте делается, как они связаны, что делать дальше;
- сам код не правит — при необходимости изменений **рекомендует открыть** Кодера/Менеджера/Ревьюера нужного проекта;
- умеет добавлять репозитории по ссылке (см. ниже).

Вкладка общего менеджера сохраняется при переключении проектов.

### Добавление репозитория по ссылке

Два способа, оба клонируют в `PROJECTS_DIR/<имя>` (имя — из репозитория, в kebab-case):

1. **Кнопка** `➕ Добавить репозиторий` — ввести URL, бэкенд сделает `git clone` и зарегистрирует проект.
2. **Разговором** — попросить общего менеджера: «добавь репозиторий `<url>`» (он использует навык `add-repository`).

После клонирования проект сразу появляется в списке. Для приватных репозиториев нужен доступ
(SSH-ключ или токен) на машине, где работает бэкенд.

### Скиллы агентов

Навыки общего менеджера лежат в репозитории в `backend/skills/` (формат `.claude/skills`):

- `add-repository` — клонирование репозитория по URL;
- `inspect-projects` — обзор проектов и рекомендации, какого агента открыть.

При старте бэкенда они копируются в `<PROJECTS_DIR>/.claude/skills/`, откуда их подхватывает `claude` CLI.
Чтобы добавить новый навык — создай папку `backend/skills/<имя>/SKILL.md` и перезапусти бэкенд.

### Терминалы и запуск приложения проекта

- **⌨ Терминал** — интерактивный PTY-терминал в директории проекта (можно открыть несколько).
- **Старт :8080** — поднимает приложение проекта на порту 8080: команда подбирается из `package.json`
  (приоритет скриптов `start` → `dev` → `preview`; для Vite автоматически проставляются `--host`/`--port`),
  а если скриптов нет — папка отдаётся статикой. **Стоп** останавливает, **🌐 Открыть :8080** открывает в браузере.
