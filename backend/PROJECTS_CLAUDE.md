# Инструкции для проектов (AI Workspace)

Этот файл автоматически кладётся в корень `PROJECTS_DIR` при старте бэкенда (из
`backend/PROJECTS_CLAUDE.md`) и действует на **ВСЕ проекты**, запускаемые из веб-IDE: агенты и
терминалы работают с `cwd` внутри папки проекта, поэтому `claude` наследует этот родительский
`CLAUDE.md`. **Не редактируй вручную** — перезаписывается при каждом запуске IDE.

## Запуск приложений проекта — всегда по сетевому хосту (0.0.0.0)

IDE работает на сервере. `localhost` / `127.0.0.1` внутри сервера **не виден** из браузера или
телефона пользователя. Поэтому любой dev/preview/HTTP-сервер проекта поднимай с биндом на **все
интерфейсы (`0.0.0.0`)**, а не на localhost — тогда он доступен по сетевому IP машины (тот же IP,
на котором открыта сама IDE), например `http://192.168.1.126:5731`.

Правила:
- **Бинди на `0.0.0.0`**, не на localhost. IP в код/конфиг не хардкодь — открывай по сетевому IP машины.
- Выбирай **свободный порт** (диапазон 5174–5999, например `5731`). 🚫 НЕ `5173` (это веб-IDE) и НЕ `3001` (бэкенд IDE).
- Перед запуском проверь, что порт свободен (`ss -ltn | grep :5731`); если занят — возьми другой.
- После запуска **сообщи пользователю точный URL**: `http://<IP-машины>:<порт>`.

Шпаргалка по фреймворкам (подставь свой порт вместо `5731`):

| Стек | Команда |
|------|---------|
| Vite (dev) | `npm run dev -- --host 0.0.0.0 --port 5731` |
| Vite (preview) | `npm run preview -- --host 0.0.0.0 --port 5731` |
| Next.js | `npx next dev -H 0.0.0.0 -p 5731` |
| Create React App | `HOST=0.0.0.0 PORT=5731 npm start` |
| Node / Express | `app.listen(5731, '0.0.0.0')` |
| Python (статика) | `python3 -m http.server 5731 --bind 0.0.0.0` |
| Python (uvicorn) | `uvicorn app:app --host 0.0.0.0 --port 5731` |
| Vue CLI | `npm run serve -- --host 0.0.0.0 --port 5731` |

Если фреймворк по умолчанию слушает только localhost — обязательно добавь флаг хоста, иначе
пользователь не сможет открыть приложение.
